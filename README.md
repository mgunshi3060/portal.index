# 社内ポータル — 運用マニュアル

## ファイル構成
```
portal/
├── index.html   ← 一般ユーザー閲覧ページ（これをブラウザで開く）
├── admin.html   ← 管理者専用ページ（PDFアップロード・管理）
└── README.md    ← このファイル
```

## 初期ログイン情報（デモ用）
| 項目 | 値 |
|------|-----|
| 管理者ID | admin |
| パスワード | portal2025 |

※ ログイン後「パスワード変更」ページで必ず変更してください。

## 基本的な使い方

### PDFをアップロードする（管理者のみ）
1. `admin.html` を開いてログイン
2. 左メニュー「PDFアップロード」をクリック
3. PDFをドラッグ＆ドロップ（または「クリック」で選択）
4. カテゴリ・公開日・重要フラグを設定
5. 「アップロードして公開」ボタンを押す
→ `index.html` のお知らせページとホームに即時反映されます

### ドキュメントを削除・管理する
1. 左メニュー「ドキュメント管理」をクリック
2. ファイル名検索・カテゴリ絞り込みで目的のファイルを探す
3. 「削除」ボタンで削除 / 「⚠重要」ボタンで重要フラグ切り替え

### 重要アラートを表示する
- アップロード時に「⚠ 重要（アラート表示）」にチェックを入れる
- → ホームページの上部に赤いアラートバーが自動表示される
- ドキュメント管理画面から後から重要フラグを変更することも可能

| その他 | 上記に当てはまらないもの |

## データの保存場所
- ブラウザの **LocalStorage** に保存されます
- 同じブラウザ・同じドメインで開けば共有されます
- **注意**：ブラウザのキャッシュ削除でデータが消えます
- 本番運用ではサーバー側（Google Drive API / SharePoint 等）への保存を推奨

## GitHub Pages への公開手順
1. `portal/` フォルダをGitHubリポジトリにプッシュ
2. リポジトリ Settings → Pages → Source: main branch → /portal
3. 公開URLが発行される
4. NASのPCのブラウザでブックマーク
